# CodePilot1C EDT — макеты, навыки, память агента и автоматизация инфобаз

Сборка: `0.1.7.20260420-0315`

Крупный релиз за три месяца: агент научился генерировать печатные формы из промпта, получил систему навыков и долгосрочную память, починен tool calling для Kimi/Moonshot, закрыты 16 багфиксов EDT-инструментов и автоматизирована работа с инфобазами.

---

## Макеты 1С: создание, генерация, инспекция

Печатные формы — одна из топ-3 болей 1С-разработки. Теперь агент покрывает полный цикл.

**Создание.** Агент добавляет макет нужного типа к любому объекту: табличный документ (.mxl), схема компоновки, HTML, текстовый, бинарный.

**Генерация содержимого.** Новый инструмент `render_template` — агент описывает структуру печатной формы в JSON (секции Шапка, ШапкаТаблицы, СтрокаТаблицы, Подвал), а плагин превращает это в готовый .mxl с именованными областями, параметрами, стилями и форматированием. Один промпт — "сделай печатную форму для накладной с шапкой, таблицей товаров и итогами" — и агент генерирует готовый макет.

**Чтение.** Парный инструмент `inspect_template` — агент видит содержимое существующего макета: сетку ячеек, параметры, именованные области. Полезно когда нужно доработать существующую печатную форму, а не создавать с нуля.

## Навыки (@skill) и профили агента

Агент получил систему навыков — заранее подготовленные инструкции, которые подключаются через `@skill` в сообщении.

Готовые навыки:

- **architect** — архитектурный обзор и проектирование конфигурации
- **validator** — контроль корректности, прогон диагностик
- **explain**, **refactor**, **review** — обновлённые навыки для работы с кодом

Профили агента теперь поддерживают локальные override'ы — можно переопределить настройки под конкретный проект, не трогая основной конфиг. В UI добавлена отдельная страница для управления навыками.

---

## Долгосрочная память

Агент запоминает факты между сессиями: контекст проекта, принятые решения, особенности конфигурации. В новом разговоре память подтягивается автоматически — не нужно каждый раз объяснять агенту с чем он работает.

В Eclipse добавлено представление Memory Inspector — видно что именно агент запомнил, можно точечно отредактировать или удалить.

---

## Kimi / Moonshot: tool calling починен

Модели Kimi (Moonshot) теперь корректно вызывают инструменты. Раньше Kimi уходил в reasoning-only ответы и не дёргал tools — теперь это исправлено:

- Kimi выделен в отдельное семейство моделей, убрано наследование Qwen-специфичной логики
- Reasoning-контент сохраняется в истории (раньше терялся)
- Tool calls ищутся и в content, и в reasoning_content — Kimi иногда кладёт вызов именно туда

---

## Формы: удобнее и безопаснее

- Fuzzy-алиасы в параметрах — агент реже ошибается с названиями полей
- Ранняя валидация мутаций — агент заранее видит какие действия допустимы
- Новые операции: `add_command` и `add_button` на форме
- Защита .form файлов от прямой записи — агент не может сломать форму через WriteTool/EditFileTool, только через валидированный API

## 16 багфиксов EDT-инструментов

Пачка точечных фиксов для стабильности: защита от пустых параметров, корректная работа с synonym через EMap API, обработка разнобоя версий EDT, форсированный экспорт с таймаутом при материализации формы, переработка tool surface prompt для Qwen-моделей.

## Автоматизация инфобаз

Три закрытых issue:

**Автономный сервер** (#28) — `update_infobase` теперь работает с базами, подключёнными через автономный сервер, а не только с файловыми.

**Фоновое обновление** (#29) — на больших конфигурациях обновление уходит в фон (`async=true`), агент получает `jobId` и продолжает работу. Новый инструмент `update_infobase_status` для проверки готовности.

**Подключение инфобазы** (#30) — новый инструмент `connect_infobase`. Агент сам подключает базу к проекту: файловую или через автономный сервер. Пароли не попадают в логи, пути валидируются, замена существующей привязки требует явного `force`.

## Стабильность редактирования, профилирования и статистики

- Исправлен порядок обработки `edit_file`: агент не повторяет одно и то же редактирование до того, как увидит результат выполненного инструмента.
- Инструменты профилирования EDT доступны в нужных рабочих профилях агента и запрашивают разрешение там, где меняют состояние EDT.
- Статистика расхода токенов сохраняется при преобразовании обычных ответов провайдера в потоковый вывод чата.
