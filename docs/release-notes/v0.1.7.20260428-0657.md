# CodePilot1C EDT — hotfix: безопасный edit_file и совместимость streaming replay

Сборка: `0.1.7.20260428-0657`

Повторный хотфикс к релизу `0.1.7.20260428-0404`. В сборку включены дополнительные исправления для безопасного редактирования файлов через `edit_file`, корректной передачи replay usage в OpenAI-совместимых потоковых сценариях, совместимости Kimi и доступности профилирования после интеграции tool-surface.

---

## Безопасные fuzzy-замены в edit_file

- `edit_file` стал осторожнее при fuzzy-поиске заменяемого фрагмента: неоднозначные совпадения не применяются молча.
- Для SEARCH/REPLACE hunk добавлены проверки, которые не дают заменить не тот участок файла при близких, повторяющихся или частично совпадающих блоках.
- Публичное поведение и тесты скорректированы под безопасную модель: лучше вернуть понятную ошибку и попросить уточнить фрагмент, чем внести рискованное изменение.

---

## OpenAI-совместимый streaming/non-stream replay usage

- При replay обычного ответа в потоковый формат usage теперь прокидывается одинаково для streaming и non-stream веток OpenAI-совместимых провайдеров.
- Это сохраняет корректную статистику токенов в UI и устраняет расхождения между backend fallback и обычным streaming-путем.

---

## Совместимость Kimi

- Тестовое ожидание для Kimi выровнено с реальным контрактом `thinking.type=disabled`.
- Проверка фиксирует совместимость с провайдером без изменения поведения остальных OpenAI-совместимых интеграций.

---

## Профилирование после tool-surface интеграции

- Экспозиция профилирования остается доступной после интеграции tool-surface.
- `start_profiling` и `get_profiling_results` по-прежнему видны в рабочих сценариях, где они нужны для EDT debug и YAxUnit.
