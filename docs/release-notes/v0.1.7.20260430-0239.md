# CodePilot1C EDT — hotfix: защита edit_file от BSL-склеек

Сборка: `0.1.7.20260430-0239`

Хотфикс закрывает сценарий, в котором fuzzy-режим `edit_file` мог повредить программный модуль 1C: склеить соседние процедуры, удвоить `КонецПроцедуры` или сшить операторы на одной строке после нормализации пробелов.

---

## Исправлено

- `FuzzyMatcher` теперь строит точную карту offset'ов между нормализованным текстом и исходным файлом, вместо приближенного пересчета по строкам.
- Нормализованный поиск больше не расширяет заменяемый фрагмент на соседний BSL-код при отличиях CRLF/LF и trailing whitespace.
- Fuzzy-замены в `FileEditApplier` и прямом пути `EditFileTool` дополнительно проверяют итоговый фрагмент перед записью.
- Заблокированы очевидные BSL-склейки:
  - `КонецПроцедурыКонецПроцедуры` / `КонецФункцииКонецФункции`;
  - две декларации `Процедура` / `Функция` на одной строке;
  - `КонецПроцедуры` / `КонецФункции`, сшитый со следующей процедурой;
  - операторы, сшитые через `;\t`.
