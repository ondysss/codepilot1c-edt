# CodePilot1C EDT — аккаунт CodePilot и backend-провайдер по входу

Сборка: `0.1.7.20260318-0747`

Этот релиз добавляет встроенный аккаунт CodePilot в EDT и связывает его с transient backend-провайдером моделей. Теперь пользователь может войти или зарегистрироваться прямо из плагина, получить API key от backend-сервиса и сразу начать работу без ручной настройки OpenAI-compatible провайдера.

---

## Аккаунт CodePilot в настройках

В плагине появилась отдельная страница аккаунта:

- вход и регистрация прямо из EDT
- отображение сервера, пользователя и текущего статуса подключения
- просмотр usage-метрик: бюджет, расход, остаток и токены
- обновление ключа и выход из аккаунта

Для регистрации добавлен двухшаговый сценарий с подтверждением по email, а для существующих пользователей отдельный диалог входа.

---

## Backend-провайдер моделей

После логина или регистрации плагин поднимает transient LLM provider с id `backend` и использует backend API key без ручного создания записи в настройках провайдеров.

Что изменилось:

- `LlmProviderRegistry` теперь умеет держать runtime backend provider отдельно от сохранённых конфигураций
- активный провайдер корректно выбирается с учётом backend-входа и fallback на обычные configured providers
- при выходе backend provider очищается, а активный провайдер переключается на доступный fallback

Это делает сценарий "установил плагин -> вошёл в аккаунт -> сразу работаешь" рабочим из коробки.

## UI и совместимость

Дополнительно обновлены:

- страница настроек провайдеров
- локализованные строки и пункты UI для сценариев аккаунта
- статус и инициализация плагина при отсутствии вручную настроенного провайдера

Итог:

- регистрация и вход теперь встроены в EDT
- backend API key автоматически превращается в рабочий LLM provider
- usage и состояние аккаунта видны прямо в настройках плагина
